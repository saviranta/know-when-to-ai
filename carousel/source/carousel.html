Know when to AI

When AI isn't the answer.

A field guide to find nails that can be hammered with AI.
Most problems route elsewhere. Here is how to tell.
Page 02 · Framing

AI in organisations is a different problem.

AI for chat, search, code
The user sees the output immediately. Errors are visible in the moment. Blast radius stops at one person.
AI inside an organisation
A decision is delegated to a non-deterministic process. Consequences reach third parties. Errors hide in traces no one reads.
So
The same tools do not work. The frame changes. The method has to change with it.

Page 05 · Refusals

What the method refuses.

Refusal 1
Not a tech problem.
The fix is process, ownership, policy, or measurement. No tool helps. Naming it spares a project and a budget.
Refusal 2
Off-limits to AI.
No set of controls makes this acceptable. The decision stays with a human who can bear it, now and later.
Refusing a bad engagement is a first-class output.
